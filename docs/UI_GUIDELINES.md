# VENDOORA UI Guidelines

## Visual Direction

The UI is built around a premium, editorial, and glass-rich experience that resembles a luxury marketplace rather than a traditional SaaS dashboard. The design language mixes warm celebration aesthetics with modern, polished interface components.

## Colors

The current palette is anchored by the following brand tones:

- Electric cobalt: #1E40AF
- Sunset orange: #F97316
- Soft mint: #D1FAE5
- Indigo/purple accent: #6366F1
- Deep dark vendor dashboard tone: #0C0C0E

These appear in the landing experience and the dark vendor dashboard mode.

## Typography

- Sans: Poppins
- Display/heading: Poppins
- Serif: Playfair Display
- Monospace: JetBrains Mono

The global theme defines these for the app shell and interface.

## Spacing

The layout favors generous whitespace, large rounded surfaces, and card-based sections. Common patterns include:

- Large rounded containers with 32px radius
- Wide vertical spacing between major sections
- Mobile-first stacking with condensed padding on smaller screens

## Glassmorphism

Glass effects are a major part of the visual system:

- Frosted translucent cards
- Soft blur backgrounds
- Subtle borders and shadow layering
- Layered overlays for landing and dashboard surfaces

## Mesh Gradients

The UI uses orb-like blurred gradients for atmosphere and depth, especially on the landing page. These appear as background blobs rather than heavy full-screen gradients.

## Cards

Cards are used for:
- Vendor discovery
- Category tiles
- Marketplace products
- Inventory management
- Notification surfaces
- Dashboard panels

The design system favors rounded corners, soft borders, and high-contrast content blocks.

## Buttons

Buttons follow a simple visual hierarchy:

- Primary actions use cobalt or purple emphasis
- Secondary or neutral actions use lighter surfaces
- Vendor dashboard actions use higher-contrast dark surfaces

## Animations

Motion is used for:
- Hover states
- Transitions between modes
- Smooth scrolling to sections
- Reels-style interactions
- Dashboard microinteractions

## Responsive Rules

The current experience is designed to scale from mobile to large screens:

- Mobile: stacked sections, compact navigation, single-column cards
- Tablet: adaptive grids for vendors and categories
- Desktop: wider content columns and richer side-by-side layouts

## Vendor Dashboard

The vendor experience uses a darker palette and more structured panels for:

- Analytics and progress indicators
- Booking management
- Inventory controls
- Marketplace publishing
- Notifications
- Reels publishing

## Landing Page

The consumer landing experience emphasizes:

- Clear hero messaging
- Search and category discovery
- Vendor cards and featured providers
- Reels and marketplace content blocks
- Mobile-friendly bottom navigation patterns
