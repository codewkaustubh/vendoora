# VENDOORA Development Roadmap

## Phase 1 - Setup

- Finalize local environment setup
- Document environment variables and startup steps
- Standardize development practices for frontend and backend work

## Phase 2 - Database

- Complete Prisma schema alignment with backend controllers
- Add migrations for missing fields and relationships
- Introduce stronger constraints and indexes for search and booking workflows

## Phase 3 - Authentication

- Implement a production-ready auth experience
- Add Google OAuth support
- Harden JWT handling and account recovery flows

## Phase 4 - Marketplace

- Expand marketplace listing flows and discovery views
- Improve filtering, search, and availability rules
- Support richer product metadata and image handling

## Phase 5 - Booking

- Formalize inquiry-to-booking lifecycle
- Introduce richer status tracking and notifications
- Support customer and vendor booking histories

## Phase 6 - Vendor Dashboard

- Improve analytics, inventory, and bookings management
- Add deeper vendor profile controls and operational dashboards
- Refine UI flows for high-frequency actions

## Phase 7 - Payments

- Introduce secure payment and payout flows
- Add booking deposits, settlement states, and invoice handling

## Phase 8 - AI

- Add AI pricing intelligence and vendor matchmaking
- Improve content generation and amendment support for reels and listings
- Expand recommendation systems

## Phase 9 - Deployment

- Prepare production deployment on Vercel, Render, or Railway
- Add staging and production environment management
- Validate database migrations and observability processes
